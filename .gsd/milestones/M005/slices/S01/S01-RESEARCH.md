# S01: Contract Rebuild & Test Fixup — Research

**Date:** 2026-03-23
**Depth:** Light — mechanical fix with known patterns

## Summary

The contract already has `metadata_uri: String` as the last parameter on both `create_payment` (4 args: payment_id, total_amount, operator, metadata_uri) and `create_milestone_payment` (4 args: payment_id, total_amount, milestone_count, metadata_uri). The test file at `tests/gherkin-pay.ts` has 6 call sites that need updating — 5 `createPayment` calls missing the 4th arg and 1 `createMilestonePayment` call missing the 4th arg.

No build artifacts exist in this worktree (`target/types/` and `target/idl/` are empty/missing), so `anchor build` must run first to generate the IDL and types that the tests import. Anchor CLI 0.31.1 matches the Cargo.toml dependency version, so no version mismatch is expected.

## Recommendation

1. Run `anchor build` to generate IDL and types
2. Add `metadata_uri` string argument to all 6 test call sites
3. Run `anchor test` to verify all existing suites pass

This is a straightforward parameter-threading fix with no architectural decisions needed.

### Build Order

1. **`anchor build`** — must succeed first; generates `target/types/gherkin_pay.ts` and `target/idl/gherkin_pay.json` that the test file imports. This retires the "Anchor build compatibility" risk from the roadmap.
2. **Thread `metadata_uri` through all 6 test call sites** — pure string addition, no logic changes. Use a test URI like `"https://example.com/travel-rule/test"`.
3. **`anchor test`** — runs the full suite on localnet validator. All 6 describe blocks must pass.

### Verification Approach

- `anchor build` exits 0 — confirms contract compiles with metadata_uri
- `grep -c metadata_uri target/types/gherkin_pay.ts` returns ≥ 1 — confirms types include the field

## Constraints

- Anchor CLI 0.31.1 matches `Cargo.toml` dependency `anchor-lang = "0.31.1"` — no version mismatch expected
- Two programs in workspace (`gherkin-pay`, `gherkin-pay-hook`) — both must build
- Mock Pyth fixture at `fixtures/mock-pyth-feed.json` is loaded by test validator via `Anchor.toml` `[[test.validator.account]]` — must remain intact for S02
- `metadata_uri` is not referenced in `#[instruction()]` attributes (only `payment_id`, `total_amount`, `operator`/`milestone_count` are) — so adding it as the last instruction parameter is safe per the KNOWLEDGE.md note about `#[instruction()]`

## Common Pitfalls

- **Forgetting one call site** — there are exactly 6 call sites across 6 describe blocks. After editing, `rg "createPayment|createMilestonePayment" tests/gherkin-pay.ts` should show all 6 now have the extra arg.
- **Wrong arg position for createMilestonePayment** — the contract signature is `(payment_id, total_amount, milestone_count, metadata_uri)`, so metadata_uri goes after the count `2`, not before it.
- **Worktree missing node_modules** — per KNOWLEDGE.md, worktrees need `bun install` before build/test. Ensure `node_modules` exists or run install first.
